## üöÄ How to Contribute

### Step-by-Step Guide

1. **Choose Your Animation**

   - Pick an animation type from the list above
   - Ensure it's not already implemented
   - Plan your design and functionality

2. **Create Your Component**

   ```bash
   # Create a new folder in the animations directory
   mkdir animations/your-animation-name
   cd animations/your-animation-name
   ```

3. **File Structure**
   Create these three files in your animation folder:

   ```
   animations/your-animation-name/
   ‚îú‚îÄ‚îÄ index.html    # Main HTML structure
   ‚îú‚îÄ‚îÄ style.css     # CSS animations and styling
   ‚îî‚îÄ‚îÄ script.js     # JavaScript functionality (if needed)
   ```

4. **HTML Template**

   ```html
   <!DOCTYPE html>
   <html lang="en">
     <head>
       <meta charset="UTF-8" />
       <meta name="viewport" content="width=device-width, initial-scale=1.0" />
       <title>Your Animation Name</title>
       <link rel="stylesheet" href="style.css" />
     </head>
     <body>
       <!-- Your animated content here -->
       <script src="script.js"></script>
     </body>
   </html>
   ```

5. **CSS Guidelines**

   - Use CSS3 animations and transitions
   - Include keyframes for complex animations
   - Make it responsive for all devices
   - Use CSS custom properties (variables) when possible
   - Add smooth transitions and easing functions

6. **JavaScript Guidelines**

   - Keep code clean and well-commented
   - Use modern JavaScript (ES6+)
   - Add event listeners for interactivity
   - Optimize performance for smooth animations
   - Handle edge cases and errors gracefully

7. **Testing Your Animation**

   - Test in multiple browsers (Chrome, Firefox, Safari, Edge)
   - Check responsiveness on different screen sizes
   - Verify smooth performance and no lag
   - Test all interactive elements

8. **Documentation**
   - Add comments explaining complex animations
   - Document any special features or requirements
   - Include usage instructions if needed

### üìù Code Standards

- **HTML**: Use semantic HTML5 elements
- **CSS**: Follow BEM methodology for class naming
- **JavaScript**: Use camelCase for variables and functions
- **Comments**: Add meaningful comments for complex logic
- **Indentation**: Use 2 or 4 spaces consistently

### üé® Design Guidelines

- **Color Scheme**: Use modern, harmonious color palettes
- **Typography**: Choose readable and stylish fonts
- **Layout**: Ensure clean, organized layouts
- **Animations**: Keep animations smooth and purposeful
- **Accessibility**: Consider users with disabilities

### üîÑ Submission Process

1. **Test Thoroughly**

   - Cross-browser compatibility
   - Mobile responsiveness
   - Performance optimization

2. **Update Main Index** (if applicable)

   - Add your animation to the main navigation
   - Include a preview or description

3. **Commit Your Changes**

   ```bash
   git add .

   ```

4. **Create Pull Request**
   - Write a clear PR title and description
   - Include screenshots or GIFs of your animation
   - Explain what your animation does and how it works
